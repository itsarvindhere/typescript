Let's say we have a function -

    const inferItemLiteral = <T>(t: T) => {
        return {
            output: t
        };
    };

This is a generic function that takes a type parameter 'T' and returns an object with a property 'output' of type 'T'.

We can call it like -

    const result1 = inferItemLiteral("hello world");
    const result2 = inferItemLiteral(123);

But, as we learnt in the previous sections, TypeScript will infer the type of 'T' as 'string' for 'result1' and 'number' for 'result2'. This means that the type of 'output' in 'result1' will be 'string' and in 'result2' it will be 'number'.

Moreover, we also want in this case that we should only be able to pass a 'number' and a 'string' to the function. If we try to pass any other type, it should give us an error. 

Now, we know from the previous sections that to infer a literal type, we can use a constraint on the type parameter 'T'. We can do this by using the 'extends' keyword -

    const inferItemLiteral = <T extends string>(t: T) => {
        return {
            output: t
        };
    };

Here, we have constrained 'T' to be a subtype of 'string'. This means that 'T' can only be a string literal type. Now, if we try to call the function with a number, it will give us an error -

    const result1 = inferItemLiteral("hello world"); // result1 will have type { output: "hello world" }

But, it starts showing an error when we try to call it with a number -

    const result2 = inferItemLiteral(123); // Error: Argument of type 'number' is not assignable to parameter of type 'string'.

How do we tell TypeScript that the type parameter 'T' can be either a 'string' or a 'number'? We can use a union type for this -

    const inferItemLiteral = <T extends string | number>(t: T) => {
        return {
            output: t
        };
    };

Now, we can call the function with both a string and a number without any errors. Moreover, if you try to call it with any other type, it will give you an error -

    const result1 = inferItemLiteral("hello world"); // result1 will have type { output: "hello world" }
    const result2 = inferItemLiteral(123); // result2 will have type { output: 123 }

    const result3 = inferItemLiteral(true); // Error: Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
