When it comes to type systems, there are two different types of type systems - 

    1. Structural Type System
    2. Nominal Type System
   
# STRUCTURAL TYPE SYSTEM

Structural Type System is a type system where the type compatibility is based on the structure of the types. In this type system, if two types have the same structure, they are considered to be compatible. TypeScript uses a structural type system.

For example, let's say we have this -

    class Car {
        make: string;
        model: string;
        year: number;
        isElectric: boolean;
    }
 
    class Truck {
        make: string;
        model: string;
        year: number;
        towingCapacity: number;
    }
 
    const vehicle = {
        make: "Honda",
        model: "Accord",
        year: 2017,
    }

Here, we have two classes and an object. The 'Car' class has a structure that includes 'make', 'model', 'year', and 'isElectric' properties. The Truck class has a structure that includes 'make', 'model', 'year', and 'towingCapacity' properties. The vehicle object has a structure that includes 'make', 'model', and 'year' properties.

So, if we have a function that expects an object that has 'make', 'model', and 'year' properties, -

    function printCar(car: {
        make: string
        model: string
        year: number
    }) {
        console.log(`${car.make} ${car.model} (${car.year})`)
    }

If you think about it, then all three - Car, Truck, and vehicle - have the same structure as the function parameter. So, all three are compatible with the function parameter. Therefore, we can call the function with any of these three -
 
    printCar(new Car()) // Fine
    printCar(new Truck()) // Fine
    printCar(vehicle) // Fine

This is what a Structural Type System is.

# NOMINAL TYPE SYSTEM

Nominal types are all about 'names'. So, in Nominal Type Systems, two types are considered to be compatible if they have the same name or there is an explicit declaration that they are compatible.

So, unlike the Structural Type System, in a Nominal Type System, the structure of the types does not matter. Even if two types have the same structure, they are not considered to be compatible unless they have the same name or there is an explicit declaration that they are compatible.

Some examples of languages that use Nominal Type Systems are Java, C#, Rust, Kotlin, and Swift.

For example, in these languages, if we have two classes with the same structure but different names, they are not considered to be compatible.

If in Java, we have -

    class Car {
        String make;
        String model;
        int year;
        boolean isElectric;
    }

    class Truck {
        String make;
        String model;
        int year;
        double towingCapacity;
    }

    class Vehicle {
        String make;
        String model;
        int year;
    }

    public class Main {

        static void printCar(Car car) {
            System.out.println(car.make + " " + car.model + " (" + car.year + ")");
        }

        public static void main(String[] args) {
            printCar(new Car());    // Fine, it's a Car
            printCar(new Truck());  // Compile error - Truck is not a Car
            printCar(new Vehicle()); // Compile error - Vehicle is not a Car
        }
    }

Truck and Vehicle both have make, model, and year — exactly what printCar needs — but Java doesn't care. They're not a Car, so they're rejected.

So, one way to make them compatible is to explicitly declare their relationship. We can do this by having a parent class or an interface that they both implement. For example, we can have a parent class called 'Vehicle' and have both 'Car' and 'Truck' extend it. Then, we can change the function parameter to be of type 'Vehicle' instead of 'Car'. This way, both 'Car' and 'Truck' will be compatible with the function parameter.

    class Vehicle {
        String make;
        String model;
        int year;
    }

    class Car extends Vehicle {
        boolean isElectric;
    }

    class Truck extends Vehicle {
        double towingCapacity;
    }

    public class Main {

        static void printCar(Vehicle vehicle) {
            System.out.println(vehicle.make + " " + vehicle.model + " (" + vehicle.year + ")");
        }

        public static void main(String[] args) {
            printCar(new Car());    // ✅ Fine, it's a Vehicle
            printCar(new Truck());  // ✅ Fine, it's a Vehicle
            printCar(new Vehicle()); // ✅ Fine, it's a Vehicle
        }
    }
