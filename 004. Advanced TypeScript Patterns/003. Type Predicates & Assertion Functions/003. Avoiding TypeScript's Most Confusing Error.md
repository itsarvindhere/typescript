With Assertion Functions, there is one very annoying error that you may face.

In the last section, we had this function -

    function assertUserIsAdmin(user: NormalUser | AdminUser): asserts user is NormalUser {
        if (user.role !== "admin"){
            throw new Error("Not an admin user!");
        }
    }

    const example = (user: NormalUser | AdminUser) => {
        assertUserIsAdmin(user);
        console.log(user);
    }

And this was working just fine.

Now, what if instead of a function declaration, we have a function expression -

    const assertUserIsAdmin = (user: NormalUser | AdminUser): asserts user is AdminUser => {
        if (user.role !== "admin"){
            throw new Error("Not an admin user!");
        }
    }

At first glance, it looks like nothing changed as all we did was change the function declaration to a function expression.

But as soon as we change the function declaration to a function expression, we start getting an error where we use this function -

    const example = (user: NormalUser | AdminUser) => {
        assertUserIsAdmin(user); // ERROR
        console.log(user);
    }

The error will say -

    Assertions require every name in the call target to be declared with an explicit type annotation.

Well, this is a well-known error in TypeScript and it happens because assertion functions do not work with 'Arrow Functions'. They only work with function declarations. So, if you want to use assertion functions, you have to use function declarations and not function expressions.

This may or may not get fixed in future versions of TypeScript but for now, if you want to use assertion functions, you have to use function declarations.
