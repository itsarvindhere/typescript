In the last section, we saw how we can combine Branded Types with Type Predicates.

Well, we can do the same with Assertion Functions. Let's take the same example but this time, instead of our function returning a boolean, let's say it throws an error if the password is invalid -

    function assertIsValidPassword (values: PasswordValues) {
        if (values.password !== values.confirmPassword) {
            throw new Error("Passwords do not match");
        }
    }

Remember that assertion functions only work when we use function declarations. If we try to use an assertion function with a function expression, it will not work as expected. For example.

So, right now, we will again get an error when we try to use it like this -

    const createUserOnApi = (values: Valid<PasswordValues>) => {
        // API call to create user
    }

    const onSubmitHandler = (values: PasswordValues) => {
        assertIsValidPassword(values);
        createUserOnApi(values); // ERROR
    }

The error happens because the 'createUserOnApi' function expects a 'Valid<PasswordValues>'. We know that if 'assertIsValidPassword' does not throw an error, then 'values' is of type 'Valid<PasswordValues>'. However, TypeScript does not know this because it does not understand the control flow of the assertion function.

We want to tell TypeScript that if assertIsValidPassword does not throw an error, then 'values' is of type 'Valid<PasswordValues>'. To do this, we can use the Assertion Function syntax in our function declaration -

    function assertIsValidPassword (values: PasswordValues): asserts values is Valid<PasswordValues> {
        if (values.password !== values.confirmPassword) {
            throw new Error("Passwords do not match");
        }
    }

Here, we are telling TypeScript that if this function does not throw an error, then 'values' is of type 'Valid<PasswordValues>'. Now, when we use this function in our 'onSubmitHandler', TypeScript will understand that after the assertion function is called, 'values' is of type 'Valid<PasswordValues>'.

So, this is how we can combine Branded Types with Assertion Functions to achieve better type safety in our code.
