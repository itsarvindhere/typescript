So far, we have seen how we can use 'declare global' to add types to the global scope. However, so far, we have done this only to add some types to the other libraries or packages that we are using. But what if we want to add some types to the global scope that are specific to our application?

Let's say we have a global interface -

    declare global {
        interface DispatchableEvent {
            LOG_IN: {
                username: string;
                password: string;
            }
        }


        type UnionOfDispatchableEvents = {
            [K in keyof DispatchableEvent]: {
                type: K;
            } & DispatchableEvent[K];
        }[keyof DispatchableEvent];
    }

This is a global interface that we have defined for our application. It has a single property 'LOG_IN' which is an object with 'username' and 'password' properties. We have also defined a type 'UnionOfDispatchableEvents' which is a union of all the events that we can dispatch in our application.

Do not get confused with the syntax of the 'UnionOfDispatchableEvents' type. It is a mapped type that iterates over all the keys of the 'DispatchableEvent' interface and creates a new type for each key. The new type has a 'type' property which is the key itself and all the properties of the original event.

Basically, if we have the 'DispatchableEvent' interface  like this -

    interface DispatchableEvent {
        LOG_IN: {
            username: string;
            password: string;
        },
        SIGN_UP: {
            email: string;
            password: string;
        }
    }

Then the 'UnionOfDispatchableEvents' type will be equivalent to this -

    type UnionOfDispatchableEvents = ({
        type: "LOG_IN";
    } & {
        username: string;
        password: string;
    }) | ({
        type: "SIGN_UP";
    } & {
        email: string;
        password: string;
    })

So, it just creates a new type for each event in the 'DispatchableEvent' interface and combines them into a union type. This way, we can easily create a type that represents all the possible events that we can dispatch in our application.

Anyways, let's now say that in our own application, we have this code in some other file -

    export const handler = (event: UnionOfDispatchableEvents) => {
        switch (event.type) {
            case "LOG_OUT": // ERROR
                console.log("LOG_OUT");
                break;
            case "UPDATE_USERNAME": // ERROR
                console.log("UPDATE_USERNAME", event.username);
                break;
        };
    }

So here, we have a handler function that takes an event of type 'UnionOfDispatchableEvents'. We can use the 'type' property of the event to determine which event it is and then access the properties of that event accordingly.

The issue right now is that the 'LOG_OUT' and 'UPDATE_USERNAME' events are not defined in the 'DispatchableEvent' interface, so we get an error when we try to use them in the switch statement. This is because the 'UnionOfDispatchableEvents' type only includes the events that are defined in the 'DispatchableEvent' interface.

How do we fix this?

This is where we have to use 'declare global' in this specific file to add some new events to the 'DispatchableEvent' interface. We can do this like this -

    declare global {
        interface DispatchableEvent {
            LOG_OUT: {};
            UPDATE_USERNAME: {
                username: string;
            }
        }
    }

Here, we are using 'declare global' to add two new events to the 'DispatchableEvent' interface. Now, when we use the 'UnionOfDispatchableEvents' type, it will include these new events as well, and we will not get any errors when we try to use them in the switch statement.

Remember that when we define an interface again, it does not overwrite the previous definition of the interface. Instead, it merges the new properties with the existing properties of the interface. So, in this case, we are adding new events to the 'DispatchableEvent' interface without affecting the existing events that are already defined in it.

A benefit of this approach is that we do not even need to import the 'DispatchableEvent' interface in this file to add new events to it. We can simply use 'declare global' to add new events to the global scope, and they will be available throughout our application without needing to import anything.

If you are working with a library or a package that has a global interface defined, your users can also use 'declare global' to add new properties or events to that interface without needing to even import that interface.
