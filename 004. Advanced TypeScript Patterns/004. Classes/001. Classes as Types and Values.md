Alright, let's shift our focus to some Typescript Patterns with Classes.

In TypeScript, one thing that is interesting about classes is that they can be used both as types and as values. This means that you can use a class to define the shape of an object, and you can also create instances of that class.

For example, let's say we have a class 'CustomError' -

    class CustomError extends Error {
        constructor(message: string, public code: number) {
            super(message);
            this.name = 'CustomError';
        }
    }

This is a common way to create a custom error class in TypeScript. The 'CustomError' class extends the built-in 'Error' class and adds a 'code' property.

Let's say we want to create a method to handle an error -

    const handleCustomError = (error: unknown) => {
        console.log(error.code);
    }

Ideally, we would want this method to only handle the 'CustomError' type of errors, but since we have defined the parameter as 'unknown', we cannot access the 'code' property directly.

So, how do we tell TypeScript that the 'error' parameter is of type 'CustomError'?

Well, as mentioned above, classes can be used as types. So, we can simply change the parameter type to 'CustomError' -

    const handleCustomError = (error: CustomError) => {
        console.log(error.code);
    }
