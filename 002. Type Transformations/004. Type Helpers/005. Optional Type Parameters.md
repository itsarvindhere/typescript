Alright, so we have seen how we can have multiple type parameters in a Type Helper. But, sometimes, we may not want to always pass all the type parameters.

What if we have a Type Helper that takes two arguments but we also want it to work with just one argument?

In case of functions in TypeScript, we can make the parameters optional by using the '?' operator. But, how do we do that in case of Type Helpers?

Our Type Helper right now looks like this -

    type CreateDataShape<TData, TError> = {
        data: TData;
        error: TError;
    };

Right now, we have to always pass both the type parameters 'TData' and 'TError' when using the 'CreateDataShape' Type Helper.

We cannot call it like -

    type DataShape = CreateDataShape<string>;

This will show an error -

    Generic type 'CreateDataShape' requires 2 type argument(s)

So, how do we make 'TError' optional?

We want that if the user did not specify 'TError', it should default to 'undefined'.

Well, just like functions, we can give default values to Type Parameters in Type Helpers. In this way, if the user does not pass a type for that Type Parameter, it will take the default value. We do this by using the '=' operator.

So, we can modify our 'CreateDataShape' Type Helper to give a default value of 'undefined' to 'TError' like this -

    type CreateDataShape<TData, TError = undefined> = {
        data: TData;
        error: TError;
    };

And now, if we call it with just one type argument like this -

    type DataShape = CreateDataShape<string>;

It will work perfectly fine. Here, 'TData' will be 'string' and 'TError' will take the default value of 'undefined'.

Just like functions, the default type parameters should come after the required type parameters. So, in our case, 'TError' comes after 'TData' because 'TError' has a default value while 'TData' does not. We cannot do -

    type CreateDataShape<TData = string, TError> = {
        data: TData;
        error: TError;
    };

This will give an error -

    Required type parameters may not follow optional type parameters
